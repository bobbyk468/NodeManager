I am a PhD researcher building ConceptGrade, a Knowledge Graph-based Visual Analytics
system for Explainable Automated Short-Answer Grading (ASAG). I want your critical
feedback on a proposed extension: integrating Large Reasoning Models (LRM) into the
existing pipeline.

══════════════════════════════════════════════════════════
CURRENT SYSTEM (ConceptGrade)
══════════════════════════════════════════════════════════

Architecture: Three-tier (Python backend → NestJS API → React dashboard)
LLM used: Gemini 2.5 Flash throughout

Five-stage grading pipeline:
  Stage 1 — Self-Consistent Extraction
    Gemini 2.5 Flash extracts concepts from student answer (3 passes,
    majority vote ≥2/3 for consistency)
  Stage 2 — Confidence-Weighted KG Matching
    Extracted concepts matched to rubric Knowledge Graph nodes via cosine
    similarity (high/medium/no-match thresholds → confidence weights 1.0/0.5/0.0)
  Stage 3 — LLM-as-Verifier  ← PRIMARY TARGET FOR LRM
    Gemini 2.5 Flash asked: "Is this concept chain domain-logically valid?"
    Returns {"valid": bool, "reasoning": str}
  Stage 4 — Chain Coverage Scoring
    Deterministic: coverage = matched_edges / expected_edges × 100
  Stage 5 — Final Score + Explanation
    Aggregates all signals into normalized grade [0,5] + human-readable rationale

Evaluation results (verified against production JSON files):
  Mohler 2011    (n=120, CS)          C_LLM MAE=0.3300 → C5 MAE=0.2229  −32.4%  p=0.0026
  DigiKlausur    (n=646, Neural Nets)  C_LLM MAE=1.1842 → C5 MAE=1.1262  −4.9%   p=0.0489
  Kaggle ASAG    (n=473, Elementary)   C_LLM MAE=1.2082 → C5 MAE=1.1797  +2.4%   p=0.34 (n.s.)

Dashboard: React, 11 linked/brushable charts, DashboardContext (useReducer),
           condition-gated study scaffolding (?condition=A/B)

══════════════════════════════════════════════════════════
PROPOSED LRM EXTENSION
══════════════════════════════════════════════════════════

Proposal: Replace Gemini 2.5 Flash at Stage 3 ONLY with a Large Reasoning Model.

Candidate models considered:
  1. DeepSeek-R1 (PROPOSED IDEAL)
     - Open weights → self-hostable, reproducible, no per-call API cost
     - Exposes reasoning trace via reasoning_content field
     - Comparable to o1 on domain reasoning benchmarks
     - Reasoning trace stored per answer → shown in new VerifierReasoningPanel
       component in the dashboard (direct IEEE VIS explainability contribution)

  2. Gemini 2.5 Pro with extended thinking (PRAGMATIC FALLBACK)
     - Same API as current system (zero infrastructure change)
     - thinkingConfig: { thinkingBudget: 8192 }
     - Thinking trace NOT exposed in API output → can't visualize it

  3. OpenAI o1/o3 (REJECTED)
     - Closed weights → not reproducible
     - High per-call cost for batch evaluation over 1,239 samples
     - No chain-of-thought trace in API output

Implementation:
  - Stage 3 prompt unchanged; model swapped to DeepSeek-R1
  - reasoning_content (think trace) stored in eval_results.json per answer
  - New VerifierReasoningPanel in dashboard: clicking an answer shows the
    step-by-step reasoning the LRM used to validate/reject its concept chain
  - Ablation: Stage 3 Gemini Flash vs. Stage 3 DeepSeek-R1 on Mohler 2011

Research framing:
  "LRM-augmented concept chain verification improves both grading accuracy
   and explanation quality in KG-grounded ASAG. The reasoning trace becomes
   a first-class explainability artifact visualized in the dashboard."

══════════════════════════════════════════════════════════
QUESTIONS FOR YOUR REVIEW
══════════════════════════════════════════════════════════

7. OVERALL: Any critical flaws in this proposal I have missed?
