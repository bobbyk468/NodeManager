# Feedback and Action Plan for ConceptGrade Code Review v22

I have reviewed the `conceptgrade_gemini_review_v22.md` file and analyzed the codebase. I can confirm that all seven issues outlined in the review are present in the current code. 

Here is the feedback and a proposed action plan to address each item, ordered by priority:

### **High Priority (Must fix for correctness)**
*   **Q6 — `RubricEditorPanel.tsx` (Field Name Mismatch):** 
    *   **Finding:** The `extractConceptsFromSpecs` function is incorrectly trying to read from `data.items` instead of `data.bars` for the `concept_frequency` visualization, which causes the concepts array (and `rubric_size`) to be nearly 0.
    *   **Proposed Fix:** Update the code to read from `data.bars` instead of `data.items`. This will correctly populate the concepts list and unblock the hypergeometric N calculations.

### **Medium Priority (Correctness/Data Integrity)**
*   **Q3 — `score_ablation_v2.py` (Sentinel Check Danger):**
    *   **Finding:** `os.listdir` on a restricted directory can throw a `PermissionError` and crash the script. It also blocks globally rather than by dataset.
    *   **Proposed Fix:** Wrap the directory read in a `try/except OSError` and filter `.flag` files so that it only halts if the flag matches the specific dataset being used (e.g., `ablation_checkpoint_gemini_flash_latest`).
*   **Q5 — `analyze_study_logs.py` (Inflated Dwell Times):**
    *   **Finding:** Dwell time computed using wall-clock delta can heavily inflate if a user switches tabs or leaves a session overnight.
    *   **Proposed Fix:** Introduce a cap of 10 minutes (600,000 ms) for the fallback wall-clock delta computation to prevent overnight sessions from severely skewing median dwells. Events with `capture_method == null` will still be included but bounded by the 10-minute cap.
*   **Q7 — `analyze_study_logs.py` (CSV Serialization):**
    *   **Finding:** `raw_edits` and `answer_dwells` are nested lists and are being serialized into the CSV as Python `repr` strings, which are unparseable in R/pandas.
    *   **Proposed Fix:** Remove `raw_edits` and `answer_dwells` from the main `fieldnames` exported in `write_csv()`.

### **Low Priority (Robustness & Documentation)**
*   **Q1 — `analyze_study_logs.py` (Shadowing & Closure):**
    *   **Finding:** `_median_dwell_by` shadows the outer `groups` variable and captures `all_dwells` by closure inside a loop.
    *   **Proposed Fix:** Rename the inner variable to `grps`, and extract the helper function to module level so it takes `all_dwells` as an explicit parameter, improving testability.
*   **Q2 — `analyze_study_logs.py` (Stale Docstring):**
    *   **Finding:** The docstring for `run_gap_moderation_analysis` mentions a "Session-level approximation" that was removed in v21.
    *   **Proposed Fix:** Update the docstring to correctly state that it uses exact per-edit rows while preserving the notes about the working correlation ρ and Logit link.
*   **Q4 — `visualization.service.ts` (Stat per request):**
    *   **Finding:** Calling `stat()` on every request adds overhead and introduces a Time-of-Check to Time-of-Use (TOCTOU) race condition if the file is deleted before the stat finishes.
    *   **Proposed Fix:** Wrap the `stat()` call in a `try/catch`. If `stat()` fails but a cached version exists, it will return the cached data instead of throwing an unhandled promise rejection.
